Coding Challenge

Welcome to this single-page application, built with:

  • ASP.NET Core and C# for cross-platform server-side code
  • Angular and TypeScript for client-side code
  • Bootstrap for layout and styling

In addition to this home screen, this application contains four test pages:

  • Get First City By Name: This form allows the user to find a city by the exact name. It will only return one result even if there is more than one city by that name. For instance, Des Moines has two instances in the table, but only the first is displayed (in keeping with the expected return in part one of the challenge):
    • Des Moines, IA
  • Search First City Like...: This form allows the user to find a city by a piece of their name (case sensitive). It also allows the user to input latitude and longitude, which will change the match score along with the closeness of the match. However, for every instance where there are multiple cities with the exact same name, only the first instance is returned. For instance, Des Moines has two instances and West Des Moines has one, but the first from each is displayed (in keeping with the expected return in part one of the challenge):
    • Des Moines, IA
    • West Des Moines, IA
  • Get All Cities By Name: This form allows the user to find a city by the exact name. In the case where multiple cities have the same name, all results are displayed. For instance, Des Moines has two instances:
    • Des Moines, IA
    • Des Moines, WA
  • Search City Like...: This form allows the user to find a city by a piece of their name (case sensitive). It also allows the user to input latitude and longitude, which will change the match score along with the closeness of the match. For instance, Des Moines has two instances and West Des Moines has one. All are displayed:
    • Des Moines, IA
    • Des Moines, WA
    • West Des Moines, IA

Each of these pages uses an endpoint in the backend, which can be hit directly:

  • https://localhost:5001/cities: returns one city for each city name
    • eg: "/Des Moines" returns Des Moines, IA
    • eg: "?like=Des Moines" returns Des Moines, IA and West Des Moines, IA
    • Other parameters include latitude and longitude, which will adjust the match score when used together
  • https://localhost:5001/all-cities: returns all cities for each city name (this is not RESTful, but my impression is that if this was a production environment, this endpoint would replace the previous one)
    • eg: "/Des Moines" returns Des Moines, IA and Des Moines, WA
    • eg: "?like=Des Moines" returns Des Moines, IA, Des Moines, WA, and West Des Moines, IA
    • Other parameters include latitude and longitude, which will adjust the match score when used together
